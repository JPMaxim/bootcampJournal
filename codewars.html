Sum By Factors

week 2

I wanted to try a 4 kyu Kata for a challenge and found this one called Sum By Factors.
The goal was, given an array of integers, to produce a sorted array of all prime factors and the sum of their multiples, in ascending order. For Example:

My Solution

I used for loops and modulo to find all of the factors for each given number. Consider 16:
I would iterate over the numbers 2-16 and if 16 % i == 0, then i is a factor of 16.
I then defined a function that would check for prime and used this method to find all of the prime factors, then returned the sum.

Issues I Encountered

After an afternoon and a few error codes and corrections I got to the point where I had a working piece of code in VSCode. However, when I attempted on codewars I ran into an 'Execution Timed Out' error.
The server only allows a certain amount of time for your code to execute before it cuts off, my code was too inefficient.
It took me quite some time to figure out how to optimize my algorithms. I found that there's an algorithm called The Sieve of Eratosthenes that more efficiently finds prime numbers. It does so by iteratively marking as composite(not prime), the multiples of each prime number found. In my code I assigned 50 multiples of each prime I found to an array and then used a conditional statement in order not to check for prime on any number already in the sieve array.
I also realised I could filter out a lot of unnecessary iterations on a particular function by changing the order of some code. Instead of checking if a number was prime and then checking if it was a factor, it was more optimal to check for factor first and only run those factors through the function that checks for prime.

What I learned

I think I got some valuable take-aways: Making a plan/design with some pseudocode and looking what methods and algorithms exist before diving in, to make more efficient code with less mistakes.

JavaScript Engines

Engines

A JavaScript Engine executes JavaScript Code. I've been using Node.js as the engine for executing my code within VSCode. Since I've been using Chrome, the engine that they use to execute JavaScript in the browser is V8 Engine.

Execution Context and Memory Environment: Global and Local

Walking through the example above:
The engine starts on line 1 and assigns the variable sumNum to global memory with a value of 30.
On line 3, addOne is assigned to global memory as a function.
Next on line 7, the console.log(..) is run in the Execution context.
On line 8, the engine stores newNum in global memory and calls addOne with an arguement of 4. This starts a local Execution.
The arguement, num, is stored in local memory with value 4. From line 4, result is stored in local memory with value 5.
line 5 runs return in the local execution context which returns result.
And finally back to line 8, the value of result, (5), is assigned to newNum.

Call Stack, Memory Heap, Event Loop and Callback Queue

The call stack is responsible for keeping the flow of execution for our application. Without it, JavaScript wouldn’t know what to call or when.

The heap is responsible for storing our data. This is where the memory allocation happens.

Some functions such as setTimeout() aren't JavaScript functions and are run in Web Api's. Once these have executed in the Api they come to the event loop which puts them back in the Stack to be executed in our JavaScipt once the stack is empty

Scope and Higher Order Functions

Global Scope

Variables declared outside any function or iteration have global scope. They can be used anywhere in the program.

Function Scope

Javascript has function scope, meaning each function creates a new scope. Variables declared inside a function are not accessible from outside the function.

Block Scope

ES6 introduced let and const as well as block scope. Variables declared inside a block cannot be accessed outside a block.

Higher Order Functions

Higher order functions are functions which accept a function as a paramter or which return a function.

Promises

JavaScipt has a built in function type called a promise. It essentially promsies to do something once a function has completed.

Promises have three states:

  • Pending - Currently running
  • Resolved - Finished running
  • Rejected - Failed, returns an error message to the user to explain what happened
