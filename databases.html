Database Fundamentals

Week 2

A database is a collection of organized and structured data that is stored and managed in a way that allows for efficient retrieval and modification of the data.

Key features

  • Records (rows)
  • Attributes/fields (columns)
  • Primary Key - Unique Identifier for each record of a table
  • Data Types

Common Database activities

  • Add records
  • Edit records
  • Delete records
  • Search for records that meet criteria

Like and Wildcards

The LIKE operator is used in a WHERE clause with wildcards to search for a specified pattern in a column.

In and Between

The IN operator allows you to specify multiple values in a WHERE clause.
The BETWEEN operator selects values within a given range. The values can be numbers, text, or dates.

Relational Databases

Weeks 3-5

Data Inconsistency

In flat-file databses, data can occur more than once and can lead to data inconsistency where one record of the data is changed but the other is not and they contradict.

Data Redundancy

Data occuring more than once place in a database is wasteful on storage and can lead to those inconsistencies. So we use relational databases.

Relational Databases are more flexible (easier to add and change), accurate (less redunancy and inconsistency), and secure (you can make some entities read only or limited access).

Database Relationship Types

Types of database relationship

The relationship between databases can be:

  • One to One
  • One to Many
  • Many to Many

In other words:

  • Any one record in a table relates to only 1 record in another table.
  • Any one record in a table can relate to multiple records in another table.
  • Multiple records in a table can relate to multiple records in another table.

Many to many relationships cause problems with data inconsistency, updating difficulty and repeated data. You aviod them by creating bridging tables

Normalisation

Weeks 3-5

First Normal Form

A table in first normal form follows these criteria:

  • A single cell holds strictly one value (atomicity).
  • A primary key exists for each record.
  • No duplicated rows or columns

Second Normal Form

A table in second normal form follows these criteria:

  • Meets all criteria of 1NF
  • It has no partial dependancy

Partial Dependancy is where a piece of data is not solely dependant on the primary key.

Third Normal Form

For third normal form, there must be no transitive functional dependancy. Every attribute that is not the primary key must depend on the primary key and the primary key only.

Key Terms

API

Application Programming Interfaces (APIs) act as the middle man between two software programs.
They allow a user to communicate with a backend system through a UI.

Client

The client is the person or software who uses the API. It can be a developer, for example you, as a developer, can use Twitter API to read and write data from Twitter, create a new tweet and do more actions in a program that you write. Your program will call Twitter’s API.
The client can also be a web browser. When you go to Twitter's website, your browser is the client who calls Twitter API and uses the returned data to render information on the screen.

Resource

A resource can be any object the API can provide information about. In Instagram’s API, for example, a resource can be a user, a photo, a hashtag. Each resource has a unique identifier. The identifier can be a name or a number.
