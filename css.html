Hero Img

I'm really happy with the landing on this journal site. I found a nice journal themed image and used the background shorthand selector to set it as the background-image.
I originally wanted the image to cover the entire viewport. I found this was pretty unresponsive though, forcing the image to cover an area that is of different dimensions to the image itself meant it would cut off a lot of the image and it looked pretty bad.

Eventually what I thought to do instead was set the width to 100%, and then use a JavaScript function to have the height be proportional to the width of the viewport.
So it doesn't always cover the whole viewport, but it's more responsive and always in the right proportions. See below:

On lines 2&3, the .offsetWidth property returns the viewable width of heroContainer. Then the variable heroHeight is defined as 2/3rds of the offSetWidth. Line 6 then converts this number to a string and concatenates px. And finally .style.height selects the CSS height property for heroContainer and re-assigns it.

I also added a max-height:100vh; property. If the screen is wide enough the height will go past 100vh and I didn't want that. It just cuts off some of the bottom of the image which I think looks fine.

The function runs on the loading of the page. I did this by adding a onload="setHeroHeight()" attribute to the body. One downside is you have to hit refresh for the image to resize if you manually change the window size.

Nav bar

My Nav bar uses flexbox to position the navContainer in the centre of the page and to centre align each link container and text.

I've used media queries to change the width of the navContainer so it's either 50% on a large screen, 75% on a smaller laptop , and 100% on tablet or mobile.

I set the position of the header on the homepage as absolute, and on the other pages as fixed, and set the opacity to 0.75. This means you can see behind the bar and I'm loving how it looks, especially with that landing hero image.

I used a gradient in the background, it transitions from white in the top left to grey in the bottom right. And I made a currentPageLink class to create a seperate background around the link for the page your currently on.

Transition and Animation

On the buttons such as the one at the bottom of this section, I've added some transition and animation CSS. The transition acts on the button and the anchor elements.
Triggered by the :hover selector, the button transitions from a black background to a white background with black border, and the anchor goes from color:white to colour:black; and font-weight: 900;.

The animation acts on the button > a:hover selector. I used the animation shorthand to define the name as flash, the duration as 1 second, timing-function as linear and delay as 1 second:
