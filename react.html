Components and JSX

What is a Component?

A component is an independant reusable piece of code.
They come in two types: class components and functional components.
My understanding is that class components are somewhat outdated. I'll just be writing about and implementing functional components.

What is JSX?

JSX is a syntax extension to Javascript which acts somewhat like a template language but also comes with the full power of Javascript.
JSX produces React "elements".


React renders our components to represent HTML elements (JSX).

Component Hierarchy

We can build our components in isolation, but they have to form a hierarchy with one root component when you use them

Virtual DOM

React creates a Virtual DOM, a lightweight version of the actual DOM.
When the state of our app changes, it changes the state of the virtual DOM. React compares the virtual DOM to the actual DOM and updates the actual DOM to match.

This means that you no longer have to work with the DOM API in the browser. So no more document.getElementBy... as covered in JS DOM.

If you change a component, React re-renders it and updates the DOM. This way the only thing that is re-rendered is the specific component rather than the entire application.

Using 'create-react-app'

You can use create-react-app in the terminal to create new React applications.
You first need to have it installed: npm install create-react-app
Then you can create you app: npx create-react-app app_name
To start up the React server you run npm start in the terminal.

React App File Structure

package.json

This file has the list of node dependencies which are needed.

If you are stepping away from a project, you can delete the node_modules folder so that it's not taking up unnecessaryspace on your machine.
Because package.json contains all the tools needed for the React app, you can run npm install in the console anytime you want to return to the project and it will know all that it needs to install by reading package.json.

index.html

This will be the only HTML file in the project. All of the application components are loaded into the id="root" div.

index.js

This is the Javascript file that corresponds to the index.html.

The above lines tell you that the App component will be rendered into the div element with id="root".

App.js

This is the file for the App component. It's standard to capitalize the names for components. The App acts as the container for all other components.

Props

React takes the attributes we declare when it renders a component and turns them into an object.

We refer to these objects as props, and can pass them as arguements to our components

Destructuring the variables means you don't have to keep using the props keyword everytime you use a prop variable.
You include the props in braces when you pass them as an arguement.

Passing props is a way of passing data down the hierarchy of components.

State and Methods

What is State?

State refers to data or properties that need to be tracked in an application

In functional components, we use a useState hook to track a state.
To use a useState hook in a component you have to import it into the component. You have to destructure it, (use {} on the import line), as it is a named export.

The useState function uses array destructuring, which returns two values, a variable for the current state (persons above) and a function to update the state (setPersons above).

The useState variables can also be passed down the component tree as props.

JavaScript map() method

The map() method takes each element from an original array and transforms it with a function you specify. And then outputs a new array in the same order.

So, you can use map() to iterate over an array to declare multiple instances of a component rather than calling it multiple times over multiple lines:
